# 상품 리뷰 · 상품 문의(Q&A) 기능 설계

## Context

`supabase/products.sql`에 `product_reviews`, `product_qna` 테이블과 RLS 정책, 통계 자동 갱신 트리거(`update_product_review_stats`)까지 이미 정의되어 있고, `src/hooks/useProducts.ts`에는 `fetchProductReviews`/`fetchProductQna` 읽기 전용 조회 함수가, `src/app/shop/[category]/[id]/page.tsx`에는 리뷰·Q&A 탭 UI(리스트 렌더링)까지 이미 구현되어 있다. 다만 **"리뷰 작성하기"/"문의하기" 버튼이 onClick 핸들러 없이 방치된 스텁**이라 실제 작성(INSERT)이 불가능하고, 어드민에서 리뷰를 숨기거나 문의에 답변하는 관리 화면도 전혀 없다. 이번 설계는 이 빈 구멍(작성 기능 + 어드민 관리)을 채우는 것이 목적이며, 기존 스키마·컴포넌트·훅 컨벤션을 최대한 재사용한다.

사용자 결정 사항:
- **리뷰 작성은 구매자만 가능** (`orders.status IN (paid, shipped, delivered)`으로 검증되지 않으면 작성 버튼 비활성화). `is_verified_purchase`는 항상 `true`로 기록됨 (구매자만 작성하므로).
- **리뷰에 사진 업로드 포함** — `product_reviews.images JSONB` 컬럼을 실제로 채운다. 커뮤니티 글쓰기(`src/app/community/write/page.tsx`)의 Supabase Storage 업로드 패턴(`totodo_pub_storage` 버킷)을 재사용.
- Q&A는 기존 스키마 그대로 단순 텍스트(제목+내용+비밀글 체크박스), 이미지/리치텍스트 없음.

## 1. DB 마이그레이션 — `supabase/products.sql` 하단 마이그레이션 섹션에 추가

현재 `product_reviews`/`product_qna`는 소유자(`auth.uid() = user_id`)만 `FOR ALL` 정책을 갖고 있고, 관리자용 정책이 전혀 없다 (다른 테이블은 `products_select_admin`처럼 `public.is_admin()` 기반 정책이 있음 — 이 패턴을 그대로 따른다).

```sql
-- product_reviews: 관리자 조회(숨김 리뷰 포함)/수정(숨김처리)/삭제
CREATE POLICY "product_reviews_select_admin" ON product_reviews FOR SELECT USING (public.is_admin());
CREATE POLICY "product_reviews_update_admin" ON product_reviews FOR UPDATE USING (public.is_admin());
CREATE POLICY "product_reviews_delete_admin" ON product_reviews FOR DELETE USING (public.is_admin());

-- product_qna: 관리자 조회(비공개 포함)/답변 작성/삭제
CREATE POLICY "product_qna_select_admin" ON product_qna FOR SELECT USING (public.is_admin());
CREATE POLICY "product_qna_update_admin" ON product_qna FOR UPDATE USING (public.is_admin());
CREATE POLICY "product_qna_delete_admin" ON product_qna FOR DELETE USING (public.is_admin());
```

이 정책이 없으면 어드민이 숨겨진 리뷰나 비공개 문의를 조회/답변/삭제할 수 없다 (RLS가 막음).

## 2. 유저 훅 (write 기능 추가)

### 2-1. `src/hooks/useMyProductReview.ts` (신규)

`src/hooks/useMyLectureReview.ts` 패턴을 그대로 따르되, 구매 여부 확인(`canReview`)을 추가한다.

- `fetchMyReview()`: `product_reviews`에서 본인 리뷰 조회(`maybeSingle`) + `order_items` ↔ `orders!inner(user_id, status)` 조인으로 `product_id` 구매 이력 확인 (`orders.status in ('paid','shipped','delivered')`) → `canReview` 플래그 설정. 이 조인 패턴은 `src/hooks/useMyOrders.ts`의 `order_items`/`orders` 조회 스타일을 참고.
- `submitReview({ rating, title, content, images })`: `canReview`가 false면 거부. `is_verified_purchase: true` 고정, `images`는 `[{url}]` 배열로 insert. 트리거가 `products.review_count`/`average_rating`을 자동 갱신하므로 별도 처리 불필요.
- `updateReview(...)`, 필요시 `deleteReview()`.

### 2-2. `src/hooks/useProducts.ts` 수정 (Q&A 읽기/쓰기 보강)

- `fetchProductQna`에 `userId?: string` 파라미터 추가. 현재 `.eq("is_private", false)`만 걸려 있어 **본인이 쓴 비밀글이 본인에게도 안 보이는 버그성 공백**이 있다 → 로그인 시 `.or(\`is_private.eq.false,user_id.eq.${userId}\`)`로 교체.
- `submitQna({ productId, userId, title, content, isPrivate })` 함수 추가: `product_qna`에 insert.

### 2-3. 이미지 업로드 헬퍼

리뷰 작성 폼 내부에서 `src/app/community/write/page.tsx:81-92`와 동일한 패턴으로 `totodo_pub_storage` 버킷에 `product-reviews/${userId}/${random}.${ext}` 경로로 업로드 후 `getPublicUrl` 결과를 로컬 state 배열에 누적, 제출 시 `images: string[]` 로 훅에 전달.

## 3. 유저 화면 — `src/app/shop/[category]/[id]/page.tsx` 수정

기존 스캐폴딩(활성 탭 스크롤, `StarRating` 등)은 그대로 두고 두 섹션만 확장한다.

### 리뷰 탭 (line ~590)
- `useMyProductReview(productId, user?.id)` 추가, `user` 존재 시 `fetchMyReview()` 호출 (`useEffect`).
- "리뷰 작성하기" 버튼(line 610): `!user` → `LoginModal` 오픈 (기존 패턴과 동일). 로그인했지만 `!canReview` → 버튼 비활성화 + "구매하신 분만 작성할 수 있어요" 안내. `canReview && !myReview` → 인라인 폼 토글. `myReview` 존재 시 "내 리뷰 수정"으로 라벨 전환.
- 리뷰 작성 폼은 `src/app/academy/[id]/reviews/page.tsx`의 `ReviewForm`+`StarRating`(interactive) 컴포넌트를 이식 (별점 선택 + 제목 인풋 + `textarea` 내용) + 다중 이미지 업로드 썸네일 영역(커뮤니티 글쓰기의 업로드 버튼/미리보기 UI 축소판) 추가.
- 리뷰 카드 렌더링(line 617-648)에 `review.images` 썸네일 표시 추가.
- 제출 성공 시 `fetchProductReviews` 재호출로 리스트 갱신.

### Q&A 탭 (line ~676)
- "문의하기" 버튼(line 708): `!user` → `LoginModal`. 로그인 시 인라인 폼 토글(구매 여부 무관, 로그인만 필요).
- 폼: 제목 인풋 + `textarea` 내용 + "비밀글로 작성" 체크박스 (`isPrivate`).
- Q&A 리스트에 `is_private` 항목은 이미 import돼 있으나 미사용인 `Lock` 아이콘(line 7)으로 뱃지 표시 — RLS 덕분에 리스트에 노출되는 비공개 항목은 항상 본인 것이므로 마스킹 없이 뱃지만 표시.
- 제출 성공 시 `fetchProductQna(productId, 1, user.id)` 재호출.

## 4. 어드민 훅 (신규)

### 4-1. `src/hooks/useAdminProductReviews.ts`

`src/hooks/useAdminLectureReviews.ts` 그대로 미러링 (핀 기능만 제외, `product_reviews`엔 `is_pinned` 없음):
- `fetchReviews(productId, page, keyword)` — `product_reviews` + `profiles` 별도 조회 조인 패턴 동일하게 사용.
- `toggleVisible(id, current)` — `is_visible` 업데이트.
- `deleteReview(id)`.

### 4-2. `src/hooks/useAdminProductQna.ts`

- `fetchQna(productId, page, keyword, filter: "all" | "pending" | "answered")` — `filter==="pending"`이면 `.is("answer", null)`, `"answered"`면 `.not("answer", "is", null)`.
- `answerQna(id, answerText)` — `supabase.auth.getUser()`로 관리자 id 취득 후 `{ answer, answered_at: new Date().toISOString(), answered_by: adminId }` update.
- `deleteQna(id)`.

## 5. 어드민 화면 (신규)

### `src/app/admin/shop/products/[id]/reviews/page.tsx`

`src/app/admin/academy/lectures/[id]/reviews/page.tsx`를 그대로 템플릿으로 사용 (동일한 `AdminTable`/`SearchBar`/`ResultCount`/`Pagination`/`IconActionButton`, 로컬 `StarDisplay` 컴포넌트 복제).
- COLUMNS: 작성자 / 별점 / 제목·내용 / 구매확인 / 등록일 / 상태(공개·숨김) / 관리
- 관리 버튼: 숨김 토글(Eye/EyeOff, `src/app/admin/shop/products/page.tsx:197-212`의 아이콘 토글 스타일 재사용) + 삭제(`IconActionButton` danger variant).

### `src/app/admin/shop/products/[id]/qna/page.tsx` (신규 패턴)

- `FilterTabs`(전체/답변대기/답변완료)로 `filter` 상태 제어.
- COLUMNS: 작성자 / 제목 / 비밀글 / 등록일 / 상태 / 관리.
- 행별로 "답변" 버튼 클릭 시 해당 행 아래에 인라인 확장 패널(질문 전문 + `answer` 바인딩 `textarea` + "답변 등록" 버튼)을 펼치는 `expandedId` 토글 방식 (모달 대신 — `answer`가 순수 TEXT라 리치에디터 불필요).
- 삭제는 `IconActionButton` danger variant.

### 진입 동선

`src/app/admin/shop/products/[id]/edit/page.tsx` 헤더 영역에 "리뷰 관리" / "Q&A 관리" 링크 버튼 추가 (`/admin/shop/products/${id}/reviews`, `/admin/shop/products/${id}/qna`로 이동). 강의 상세처럼 별도 `layout.tsx` 탭바를 새로 만들 필요는 없음 — 신규 서브페이지가 2개뿐이라 헤더 링크로 충분.

## 검증 방법

1. `npm run dev` 실행 후 실제 구매 이력이 있는 계정으로 `/shop/[category]/[id]` 접속 → 리뷰 작성 버튼 활성화 확인, 별점+제목+내용+이미지 업로드 후 제출 → 리스트에 반영되고 상품 상단의 `average_rating`/`review_count`가 트리거로 자동 갱신되는지 확인.
2. 미구매 계정으로 동일 페이지 접속 → 리뷰 작성 버튼이 비활성화(또는 안내 문구)되는지 확인.
3. 로그인 계정으로 Q&A "문의하기"에서 비밀글 체크 후 등록 → 본인에게는 보이고(Lock 뱃지), 다른 계정으로 접속 시 목록에서 사라지는지 확인.
4. 어드민 계정으로 `/admin/shop/products/[id]/reviews`에서 숨김 토글 시 상품 페이지 리뷰 탭에서 해당 리뷰가 사라지는지, 삭제 시 `review_count`가 감소하는지 확인.
5. `/admin/shop/products/[id]/qna`에서 답변 등록 후 상품 페이지 Q&A 탭에 "A." 답변이 노출되는지 확인.
6. `npm run lint` 통과 확인.
