# 강의 관리 고도화 플랜

## Context
현재 강의 후기(리뷰)가 강의와 독립적으로 존재하고, 관리자 메뉴가 단층 구조이며, 강의별 세부 관리 기능(프로모션/게시판/미션)이 없음. 이를 강의 중심 구조로 전면 개편한다.

**확정 사항:**
- 동영상 업로드: Supabase Storage 직접 업로드 + 유튜브 URL 입력 모두 지원 (이 경우는 목록 등에서 이름 옆에 유튜브 아이콘 표시)
- 강의 상세 탭: [정보 / 챕터 / 게시판 / 리뷰 / 과제]
- 기존 `/reviews` → `/community` 이전 (redirect 처리)

---

## 단계별 작업 계획

### Phase 1 — Foundation: DB + 커뮤니티 이전
**목적:** 후속 작업의 기반이 되는 DB 스키마 추가 및 기존 리뷰 메뉴 커뮤니티로 이전

#### 1-1. DB 스키마 추가 (`supabase/` 하단 마이그레이션 섹션)

**`supabase/academy.sql` 추가:**
```sql
-- 강의별 리뷰
CREATE TABLE IF NOT EXISTS lecture_reviews (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  lecture_id uuid REFERENCES lectures(id) ON DELETE CASCADE,
  user_id uuid REFERENCES auth.users(id) ON DELETE CASCADE,
  rating smallint NOT NULL CHECK (rating BETWEEN 1 AND 5),
  content text,
  is_hidden boolean DEFAULT false,
  is_pinned boolean DEFAULT false,
  created_at timestamptz DEFAULT now(),
  UNIQUE (lecture_id, user_id)  -- 수강자 1회 제한
);

-- 가격 프로모션
CREATE TABLE IF NOT EXISTS lecture_promotions (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  lecture_id uuid REFERENCES lectures(id) ON DELETE CASCADE,
  name text NOT NULL,           -- 슈퍼얼리버드, 얼리버드 등
  price integer NOT NULL,
  start_at timestamptz NOT NULL,
  end_at timestamptz NOT NULL,
  is_active boolean DEFAULT true,
  created_at timestamptz DEFAULT now()
);

-- 미션 (챕터별)
CREATE TABLE IF NOT EXISTS lecture_missions (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  chapter_id uuid REFERENCES lecture_chapters(id) ON DELETE CASCADE,
  title text NOT NULL,
  description text,
  due_date timestamptz,
  created_at timestamptz DEFAULT now()
);

-- 미션 제출
CREATE TABLE IF NOT EXISTS lecture_mission_submissions (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  mission_id uuid REFERENCES lecture_missions(id) ON DELETE CASCADE,
  user_id uuid REFERENCES auth.users(id) ON DELETE CASCADE,
  content text,
  file_url text,
  status text DEFAULT 'submitted',  -- submitted | reviewed | passed
  feedback text,
  created_at timestamptz DEFAULT now(),
  UNIQUE (mission_id, user_id)
);

-- lecture_board_posts에 category 컬럼 추가 (마이그레이션)
ALTER TABLE lecture_board_posts ADD COLUMN IF NOT EXISTS category text DEFAULT '일반';
-- 공지/질문/일반/칼럼 + 커스텀 가능

-- lecture_sessions에 storage path 추가 (직접 업로드용)
ALTER TABLE lecture_sessions ADD COLUMN IF NOT EXISTS video_storage_path text;
```

**RLS 정책:**
- `lecture_reviews`: 공개 read, 수강자 insert (enrollment 확인), 본인 update, admin 전체
- `lecture_promotions`: 공개 read, admin CRUD
- `lecture_missions`: 공개 read (목록), 수강자 상세 read, admin CRUD
- `lecture_mission_submissions`: 본인 read/write, admin 전체

#### 1-2. 커뮤니티 메뉴 이전

| 작업 | 파일 |

### Phase 2 — 강의별 리뷰 시스템
**목적:** 수강자만 작성 가능한 강의별 별점 리뷰 + 강의 상세 노출

#### 2-1. 훅 작성
- `src/hooks/useLectureReviews.ts` — 강의별 리뷰 목록 (핀된 것 먼저, 숨김 제외)
- `src/hooks/useMyLectureReview.ts` — 본인 리뷰 존재 여부 + 작성/수정

#### 2-2. 강의 상세 탭 개편 (`src/app/academy/[id]/layout.tsx`)
- 탭 구성: **[정보 / 챕터 / 게시판 / 리뷰 / 과제]**
- 공지 탭 제거 (게시판 카테고리 '공지'로 통합)
- 과제 탭: `lecture_enrollments` 확인 후 수강자에게만 렌더

#### 2-3. 리뷰 탭 페이지 (`src/app/academy/[id]/reviews/page.tsx`)
- 핀된 리뷰 1 ~ 2개 강의 소개 페이지 내 상단 고정 노출 (별도 카드 스타일)
- 리뷰 목록 (별점 평균 헤더)
- 수강자이고 미작성 시 리뷰 목록 최상단에 "리뷰 쓰기" 버튼 노출
- 리뷰 작성 모달 + 에디터 형식으로 (별점 선택 필수 + 텍스트)
- 리뷰는 기본적으로 시간순(desc)으로 표시되되, 본인이 작성한 리뷰는 본인한테는 최상단에 노출. 수정 버튼이 있어야 함. 다만, 편집된 경우는 (편집됨) 이라고 표시

#### 2-4. Admin 강의별 리뷰 관리 (`src/app/admin/academy/lectures/[id]/reviews/page.tsx`)
- 리뷰 목록 (작성자, 별점, 내용, 날짜)
- 핀 고정 (최대 2개 — 현재 글로벌 리뷰 5개와 별개)
- 감추기/복원 토글
- 훅: `src/hooks/useAdminLectureReviews.ts`

### Phase 3 — Admin 메뉴 체계 개편
**목적:** 단층 사이드바 → 강의 관리 하위 메뉴 구조 + 강의 상세 고정 헤더

#### 3-1. AdminLayout 사이드바 개편 (`src/components/AdminLayout.tsx`)
현재 단층 11개 메뉴 → 계층형 구조:

```
대시보드
홈 편집
메뉴 관리
회원 관리
강의 관리           ← 클릭 시 하위 펼침
  └ 강의 목록
  └ 리뷰 관리       ← 기존 admin/reviews를 여기로 이동
주문 관리
FAQ 관리
상품 관리
알림
코드 관리
```

- 하위 메뉴 있는 항목: 토글(펼침/닫힘) + active 경로 자동 펼침
- asis `admin/reviews` 경로는 `admin/academy/lectures` 하위 전용 리뷰로 교체
- asis 글로벌 리뷰(tobe 커뮤니티) 관리는 별도 유지 (사용자 확인 불필요 — 명세상 강의 하위로만 언급)

#### 3-2. 강의 상세 고정 헤더 (`src/app/admin/academy/lectures/[id]/layout.tsx` 신규)
- `[< 강의 목록]` 백버튼 + 강의명
- 하위 탭 메뉴: **[기본 정보 / 챕터·세션 / 리뷰 / 게시판 / 미션 / 프로모션]**
- `position: sticky; top: 0` 스크롤 고정
- 각 탭 → 해당 서브 라우트로 이동

현재 `src/app/admin/academy/lectures/[id]/page.tsx`에 모든 내용이 있음 → 서브 라우트로 분리:
- `page.tsx` → 기본 정보 + 챕터/세션 (기존 유지, 레이아웃 헤더 추가)
- `reviews/page.tsx` — Phase 2에서 작성
- `board/page.tsx` — Phase 6에서 작성
- `missions/page.tsx` — Phase 7에서 작성
- `promotions/page.tsx` — Phase 4에서 작성

### Phase 5 — 강의 동영상 업로드
**목적:** 세션 편집 시 파일 직접 업로드 또는 URL 입력, 자동 길이 추출

#### 5-1. 업로드 컴포넌트 (`src/components/admin/molecules/VideoUploader.tsx`)
- 탭: [파일 업로드] / [URL 입력]
- 파일 업로드: `supabase.storage.from('lecture-videos').upload(...)` → `video_storage_path` 저장
- URL 입력: 기존 `video_url` 필드 유지
- 자동 duration 추출: 업로드/URL 입력 후 hidden `<video>` 엘리먼트로 `loadedmetadata` 이벤트에서 `duration` 읽어 `duration_seconds`에 자동 입력
- 업로드는 Async Non-Blocking으로. (유튜브 업로드처럼)

#### 5-2. 세션 편집 폼에 통합
- `src/app/admin/academy/lectures/[id]/page.tsx` 내 세션 편집 부분에 VideoUploader 연결

### Phase 6 — 게시판 관리 통합
**목적:** 강의 공지 + 기존 게시판을 카테고리 통합 게시판으로

#### 6-1. 카테고리 체계
- `lecture_board_posts.category`: 공지 / 질문 / 일반 / 칼럼 (기본값)
- Admin에서 카테고리 추가/삭제 가능 → `lecture_board_categories` 테이블 또는 코드 테이블 활용

#### 6-2. Admin 게시판 관리 (`src/app/admin/academy/lectures/[id]/board/page.tsx`)
- 게시글 목록 (카테고리 필터 탭, 제목, 작성자, 날짜)
- 게시글 작성/수정/삭제
- 카테고리는 공통코드화

#### 6-3. 강의 게시판 탭 (`src/app/academy/[id]/board/page.tsx` 수정)
- 기존 "마호 칼럼" 전용 → 카테고리별 게시판으로 확장
- 목록은 모든 방문자 열람 가능
- **글 내용(상세)은 수강자만** — enrollment 확인 후 비수강자는 "수강 후 이용 가능" 안내
- 기존 `lecture_notices` 공지는 카테고리 '공지'로 마이그레이션

### Phase 7 — 미션 관리
**목적:** 챕터별 과제 출제 및 제출 관리

#### 7-1. Admin 미션 관리 (`src/app/admin/academy/lectures/[id]/missions/page.tsx`)
- 챕터별 미션 목록
- 미션 추가/수정/삭제 (제목, 설명, 마감일)
- 제출 현황 (제출자 목록, 상태 변경: submitted → reviewed → passed, 피드백 입력)
- 훅: `src/hooks/useAdminMissions.ts`

#### 7-2. 강의 과제 탭 (`src/app/academy/[id]/missions/page.tsx` 신규)
- **수강자에게만 탭 노출** (layout에서 enrollment 확인)
- 챕터별 미션 목록 + 마감일
- 제출 폼 (텍스트 + 파일 첨부)
- 제출 상태 표시
- 훅: `src/hooks/useMissions.ts`

## 검증 방법

각 Phase 완료 후:
1. `npm run build` — 타입 에러 없음 확인
2. `npm run dev` 후 브라우저 확인:
   - Phase 1: `/community` 접근, `/reviews` → redirect 확인
   - Phase 2: 강의 상세 리뷰 탭, 수강자 로그인 후 리뷰 작성
   - Phase 3: Admin 사이드바 하위 메뉴, 강의 상세 고정 헤더
   - Phase 4: 프로모션 추가 후 강의 상세 가격 변경 확인
   - Phase 5: 동영상 업로드 후 duration 자동 입력 확인
   - Phase 6: 게시판 카테고리 필터, 비수강자 내용 차단 확인
   - Phase 7: 미션 출제 → 수강자 제출 → 관리자 피드백 흐름

## 작업 순서 권고

Phase 1 → Phase 3(사이드바) → Phase 2 → Phase 4 → Phase 5 → Phase 6 → Phase 7

Phase 1·3이 기반이 되므로 먼저 진행. Phase 2~7은 독립적으로 진행 가능.
